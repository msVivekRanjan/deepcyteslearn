Module 1: Computer Architecture

CPU Fundamentals & Execution Cycle

Core Concepts

The Central Processing Unit (CPU) executes binary program instructions using the foundational Fetch, Decode, Execute, Write-Back operational pipeline cycle loop.

  • Control Unit (CU): The operational dispatcher; interprets system binary code opcodes.
  • Arithmetic Logic Unit (ALU): The functional logic core; handles mathematical operations.
  • Registers: High-speed internal memory latches (e.g., Program Counter (PC), Accumulator).
  • Clock Cycle: Microscopic oscillator synchronization pulses measuring chip step metrics.
Conveyor Assembly Analogy: The CPU runs like a hyper-speed manufacturing plant. The Program Counter is the order index tracking sheets, the Instruction Latch houses the current blueprint pattern, and the ALU is the machine press stamping values together.

↓ Interrupt Path

↓ Internal Bus
Accumulator Register (ACC) [Saves Operational Results]

Memory Fundamentals & Paging Hierarchy

Core Concepts

Computer memory maps across a tiered physical topology matrix designed to balance access velocity, data storage depth, and silicon fabric cost metrics.

  • L1/L2 Cache SRAM: Ultra-low latency memory channels mounted directly inside the physical processing core.
  • System RAM DRAM: Dynamic storage workspace containing run-time processes.
  • Virtual Memory / Paging: Kernel-managed storage allocation maps translating secondary disk spaces into logical memory configurations.
Office Desk space Analogy: L1 Cache is the document currently held in your physical hand. System RAM represents files placed flat across the desk layout. Virtual page files act like a heavy archival cabinet down the corridor.

↓ Cache Line Miss Link
DRAM Physical Main Memory RAM (~60ns Access Velocity Boundary)
↓ MMU Translation Page Fault
Secondary Page File Swap Drive (~50,000ns Access Velocity Penalty)

Storage Systems & Disk Fragmentation

Core Concepts

Persistent storage systems maintain binary arrays without active power connectivity, separating historical mechanical architectures from solid-state designs.

  • Mechanical HDD: Magnetic platters requiring mechanical actuator arms to rotate over sectors, introducing physical seek latency.
  • Solid-State SSD: Non-volatile NAND Flash memory cells that read and write blocks electronically without mechanical motion.
  • Fragmentation: Scatter-pattering where continuous files split across non-contiguous storage sectors over time, increasing system file pointer lookup overhead.

Windows Architecture & Privilege Rings

Core Concepts

The Windows NT architecture enforces structural security boundaries by separating code operational modes across processor privilege levels.

Credential Storage & Cryptographic Hashing

Core Concepts

Operating systems must avoid saving plain text authentication credentials to persistent storage media blocks.

  • One-Way Hash Functions: Passwords are run through mathematical hashing algorithms (e.g., bcrypt, SHA-256) to generate unique, fixed-length strings.
  • Salt Strings: Random data parameters are appended to passwords before hashing to protect against pre-computed table lookups.

Memory Protection Matrix (DEP & ASLR)

Core Concepts

Advanced operating system kernels protect physical volatile memory lines from code-execution exploits by utilizing DEP and ASLR frameworks.

  • DEP (Data Execution Prevention): Marks storage data blocks (like the heap and stack layouts) as strictly non-executable, preventing raw text data injections from spawning system shells.
  • ASLR (Address Space Layout Randomization): Shuffles the memory address layouts of key program segments randomly at boot time, preventing reliable exploit target pointing.
