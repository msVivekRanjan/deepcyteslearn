// PORT EXPOSURE ANALYSIS
🔴 Port 22 — SSH
▶
Secure Shell. Exposed SSH with password auth is a prime brute-force target. Look for: default credentials, old OpenSSH versions, key files in public repos.
CRITICAL EXPOSURE
🔴 Port 21 — FTP
▶
File Transfer Protocol. Often left with anonymous access enabled. Transmits credentials in plaintext. A goldmine for file exfiltration.
HIGH RISK
🔴 Port 3389 — RDP
▶
Remote Desktop. Direct GUI access to Windows systems. BlueKeep and similar vulnerabilities have made exposed RDP one of the most dangerous attack surfaces.
CRITICAL
🟡 Port 80/443 — HTTP/HTTPS
▶
Web services. Check for: admin panels (/admin, /wp-admin), directory listing, outdated CMS versions, exposed Git repositories, API endpoints.

// OPSEC FAILURE SCENARIOS

Real-world examples of investigators being burned:

💀 Browser Metadata Leak
▶
Investigator used a personal browser to access target's site. Browser fingerprint (screen resolution, fonts, timezone, plugins) matched their real identity profile — exposing them without cookies or login.
💀 Image Metadata (EXIF) Exposure
▶
Uploaded a screenshot to report. Image EXIF data contained original filename, device model, and GPS coordinates from a home device. Target was notified by a third party.
💀 Sock Puppet Account Linked
▶
Created a fake LinkedIn account but logged into it from the same IP as the real account. LinkedIn's security system linked both accounts and notified the target organization's security team.
💀 DNS Leak Through VPN
▶
Operator used a VPN for traffic routing but didn't configure DNS-over-HTTPS. DNS queries resolved through ISP's servers, revealing real IP address and investigation target to the ISP.
