Key Notes Separation of Duties (SoD) prevents a single user from holding conflicting roles (e.g., "Approve Payment" + "Initiate Payment"). Role mining tools analyse existing access patterns to suggest appropriate roles. Periodic access reviews (recertification) ensure role assignments remain valid.

Key Notes TOTP codes are valid for ~30 seconds and generated using HMAC-SHA1 with a shared secret. FIDO2/WebAuthn hardware keys are phishing-resistant — the strongest MFA form. Adaptive MFA evaluates device fingerprint, location, time of day, and behaviour anomalies to determine risk score.

Key Notes SAML 2.0 uses XML-signed assertions and is common in enterprise. OIDC uses JWT tokens over HTTPS and is popular for modern web/mobile apps. The SP never sees the user's IdP password — it only trusts the signed assertion. Single logout (SLO) must also be implemented to terminate sessions across all SPs simultaneously.

Key Notes Always use LDAPS (port 636) or STARTTLS to encrypt credentials in transit. Anonymous binds should be disabled in production. Search filters like (&(objectClass=user)(uid=jsmith)) can be injected if not sanitised — always escape user input. Connection pooling improves performance for high-volume lookups.

Key Notes Admins should never know the actual privileged password — the PAM vault injects it transparently. Session recordings are immutable audit evidence. Automatic password rotation after each session prevents credential reuse. JIT access windows should be as short as technically feasible (typically 1–4 hours).

Key Notes Zero Trust is a philosophy, not a product. Microsegmentation limits lateral movement if a breach occurs. Device compliance checks include: OS patch level, antivirus status, disk encryption, MDM enrollment, and certificate validity. Continuous monitoring uses UEBA (User and Entity Behaviour Analytics) to detect anomalies in real time after access is granted.
