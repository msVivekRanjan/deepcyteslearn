01
OSI Reference

The OSI Model

The OSI (Open Systems Interconnection) model is a 7-layer conceptual framework that explains how data travels across a network. Think of it as a recipe — each layer has a distinct job, and together they get your data from A to B.

💡 TipClick any layer below to see more details.
7
Application
HTTP, DNS, SMTP, FTP, SSH — user-facing protocols
Data
This is where users interact with network services. Protocols like HTTP (web), DNS (name resolution), SMTP (email), FTP (file transfer), and SSH (secure shell) all live here. When you type a URL, this layer kicks things off.
6
Presentation
Encryption, compression, data formatting
Data
The "translator" layer. It converts data into a format the application can understand. TLS/SSL encryption happens here. Also handles compression (like JPEG, MP3) and character encoding (UTF-8).
5
Session
Manages dialogs, sessions, checkpoints
Data
Controls when sessions start, continue, and end. Think of it like a phone call manager — it sets up, coordinates, and tears down connections between applications. Examples: NetBIOS, RPC, SQL sessions.
4
Transport
TCP (reliable) / UDP (fast) — end-to-end delivery
Segment
Breaks data into segments and reassembles them at the other end. TCP guarantees delivery with acknowledgements and retransmissions. UDP skips the handshaking — great for video streaming, gaming, VoIP where speed > reliability.
3
Network
IP addressing, routing, path selection
Packet
Handles logical addressing (IP) and routing between different networks. Routers operate here. This layer decides the best path for your packet to travel — it could go through 15 hops before reaching its destination.
2
Data Link
MAC addresses, frame delivery, error detection
Frame
Handles physical addressing using MAC addresses for delivery within a single network. Switches operate here. Wraps packets into frames, detects (and sometimes corrects) transmission errors using CRC checksums. Split into LLC and MAC sublayers.
1
Physical
Cables, signals, bits — raw transmission media
Bit
The actual physical medium — copper cables, fiber optic, Wi-Fi radio waves. Deals with voltage levels, pin layouts, timing, and transmission rates. Nothing logical here — just raw 1s and 0s moving as electrical, optical, or radio signals.
🧠 Memory Trick"All People Seem To Need Data Processing" — Application, Presentation, Session, Transport, Network, Data Link, Physical (top to bottom).
02
Protocol Stack

TCP/IP Model

The TCP/IP model is what the Internet actually runs on. More practical than OSI, it condenses 7 layers into 4, mapping closely to how real protocols work.

TCP/IP Layer OSI Equivalent Function Examples
Application Layers 5, 6, 7 User-facing protocols HTTP, DNS, SMTP, FTP, SSH
Transport Layer 4 Reliable / fast delivery TCP, UDP
Internet Layer 3 IP addressing and routing IPv4, IPv6, ICMP
Link Layers 1, 2 Hardware + local delivery Ethernet, Wi-Fi, ARP
Key differenceOSI is a conceptual model great for learning and troubleshooting. TCP/IP is the real-world implementation that powers the Internet.

Layer 3

Routing

Routing is the process of moving packets between different networks. Routers look at destination IP addresses and consult their routing tables to decide where to send traffic next.

TypeHow it worksBest forProtocols
Static Manually entered by admin Small, simple, stable networks —
Dynamic Routers learn paths automatically Large, complex, changing networks RIP, OSPF, BGP, EIGRP
Default Catch-all for unknown destinations Edge routers to ISP 0.0.0.0/0
BGP — The Internet's Routing ProtocolBGP (Border Gateway Protocol) is what connects all the world's ISPs and networks. It's often called "the protocol that runs the Internet." Every time you load a webpage, BGP decisions were made about how to route your packets.
07
Layer 2

Switching

Switches are the workhorses of local area networks (LANs). Unlike hubs, they intelligently forward frames only to the destination device, not everyone.

How a Switch Learns
1. A frame arrives on port 3 from device A (MAC: AA:BB:CC)
2. Switch records: "MAC AA:BB:CC lives on port 3" in its MAC table
3. Looks up the destination MAC — found on port 7
4. Forwards frame only to port 7 (not all ports)
5. If MAC is unknown → floods to all ports, then learns from reply

08
Translation

NAT

Network Address Translation (NAT) lets many devices on a private network share one public IP address. It's why your home router can connect 20 devices to the Internet with just one public IP.

TypeMappingUse Case
Static NAT 1 private ↔ 1 public Servers that need consistent public access
Dynamic NAT 1 private → 1 of pool Organizations with a block of public IPs
PAT / NAT Overload Many private → 1 public (via ports) Home routers, most businesses
🔒 NAT ≠ FirewallNAT makes internal IPs harder to target from outside, but it's not a security tool. You still need a proper firewall. NAT breaks end-to-end connectivity — it's a workaround, not a design goal.

11
Auto Config

DHCP

DHCP automatically hands out network settings to devices when they join. Without it, you'd have to manually set IP addresses on every device — imagine doing that in a 500-device office.

D
Discover
Client broadcasts: "Is anyone a DHCP server?"
O
Offer
Server replies: "Here's an IP you can use"
R
Request
Client says: "Yes please, I'll take that one"
A
Acknowledge
Server confirms: "It's yours for [lease time]"
DHCP ReservationsYou can configure a DHCP server to always assign the same IP to a specific device (matched by MAC address). This gives you the convenience of DHCP with the predictability of static IPs — great for printers and servers.

📖 Quick Glossary

IP Address
Logical address identifying a device on a network
MAC Address
Hardware address burnt into a NIC; used for local delivery
Packet
Data unit at the Network layer (Layer 3)
Frame
Data unit at the Data Link layer (Layer 2)
Segment
Data unit at the Transport layer (TCP)
Router
Device connecting and routing between different networks
Switch
Device forwarding frames within a LAN by MAC address
Broadcast
Traffic sent to all devices in a network segment
Default Gateway
Router a device uses to reach outside its own network
Lease
Temporary DHCP assignment of network settings
TTL
Time To Live — limits packet hops; also DNS cache duration
Dual Stack
Running IPv4 and IPv6 simultaneously on the same network
ARP
Address Resolution Protocol — maps IP addresses to MAC addresses
Latency
Time delay for a packet to travel from source to destination
Bandwidth
Maximum data transfer rate of a network connection
Throughput
Actual data transfer rate achieved in practice
