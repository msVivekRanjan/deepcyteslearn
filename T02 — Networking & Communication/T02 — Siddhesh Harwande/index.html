OSI Model — 7-Layer Architecture

The Open Systems Interconnection model is the universal blueprint for how network communication is structured. Click each layer to explore deeply.

📖 What is the OSI Model?
OSI (Open Systems Interconnection) is a conceptual framework developed by ISO that divides network communication into 7 distinct layers. Each layer has a specific responsibility and communicates only with the layers directly above and below it. It's NOT a physical implementation — it's a theoretical model used for designing, troubleshooting, and understanding networks. Think of it like a factory assembly line: each station adds something specific to the product before passing it along.

Module 02

TCP/IP Model — The Real Internet

The practical 4-layer model that actually runs the internet today. OSI is the theory; TCP/IP is the reality.

🌐 OSI vs TCP/IP
While OSI has 7 layers for conceptual clarity, TCP/IP consolidates these into 4 layers that map directly to real protocols in use. The internet was designed around TCP/IP, not OSI. OSI is like the architect's blueprint; TCP/IP is the actual building.

🔒 TCP — Transmission Control Protocol

TCP is connection-oriented and reliable. Before sending data, TCP performs a 3-Way Handshake:

① SYN — Client: "I want to talk. My seq=100"
② SYN-ACK — Server: "OK. My seq=200, ack=101"
③ ACK — Client: "Got it. ack=201" → Connected!
USE CASES
Web pages (HTTP/HTTPS), Email (SMTP/IMAP), File Transfer (FTP), SSH — anything needing guaranteed delivery.

⚡ UDP — User Datagram Protocol

UDP is connectionless and fast. No handshake, no guarantee, no retransmission. Just fire and forget.

➤ No connection setup — immediate sending
➤ No ACK — sender doesn't know if received
➤ Very low overhead — super fast
USE CASES
Live streaming, Video calls (Zoom/Meet), Online gaming, DNS lookups, DHCP — speed matters more than perfection.

IPv4 — 32-bit Internet Addressing

Every device needs an address. IPv4 is the classic system using 32-bit numbers written in dotted decimal notation.

📌 IPv4 Structure
An IPv4 address is 32 bits = 4 groups of 8 bits (called octets), each ranging from 0–255, written as W.X.Y.Z. Example: 192.168.1.10
⚠️ Network vs Host
An IP address has two parts: Network Part (identifies your network — like your street) and Host Part (identifies your device — like your house number). The subnet mask tells you which part is which.

Module 04

IPv6 — 128-bit Next Generation

IPv4 only has ~4.3 billion addresses and they're almost gone. IPv6 brings 340 undecillion addresses and built-in improvements.

🚀 Why IPv6?
By 2011, IANA exhausted the last IPv4 /8 blocks. IoT (billions of smart devices), mobile devices, and the growth of the internet made IPv6 inevitable. IPv6 isn't just "more addresses" — it also removes NAT, improves routing efficiency, mandates IPsec support, and eliminates broadcast (replaced with multicast/anycast).

Subnetting — Divide & Conquer

Subnetting lets you split a large IP block into smaller logical networks. Drag the slider to see subnets update live.

🧩 Why Subnet?
Without subnetting, a company with 1000 devices all in one /16 network would have 65,534 devices receiving every broadcast. That's chaos. Subnetting creates smaller, quieter, more secure segments — just like dividing a city into neighborhoods, each with its own local mail delivery.

Routing — GPS for Data Packets

Routers connect networks and decide the best path for each packet. Click Simulate to watch a packet travel.

🗺️ What is Routing?
Routing is the process of forwarding packets from a source network to a destination network, often across many intermediate networks. A router is a Layer 3 device that maintains a routing table — a map of known networks and how to reach them. When a packet arrives, the router looks up the destination IP, finds the best matching route, and forwards the packet out the appropriate interface.

Switching — Local Delivery via MAC

Switches forward frames inside a LAN using hardware MAC addresses. Watch the MAC table build itself as devices communicate.

🔌 What is Switching?
A switch is a Layer 2 device that operates within a single network segment (LAN). Unlike a router (which moves packets between networks), a switch moves frames between devices on the same network. It uses MAC addresses — unique 48-bit hardware identifiers burned into each NIC — to figure out exactly which port to send each frame to.

Module 08

NAT — Network Address Translation

NAT allows many devices with private IPs to share one public IP. The router acts as a smart translator between your private network and the internet.

🔄 What is NAT?
Without NAT, every device in your home would need a unique public IP address. Since IPv4 only has ~4.3 billion addresses and there are billions of devices, NAT solves this by allowing your entire home network (with private addresses like 192.168.x.x) to share a single public IP from your ISP. The router maintains a NAT translation table to track which internal device each external connection belongs to.

Module 09

VLANs — Virtual Network Segmentation

VLANs let you create logical network segments on the same physical switch. Click ports to assign them to different VLANs.

🏢 What are VLANs?
Imagine a company where HR, IT, and Sales all plug into the same physical switch. Without VLANs, all traffic is shared — everyone can see everything. With VLANs, the switch logically divides into separate broadcast domains. HR traffic stays in VLAN 10, IT in VLAN 20, Sales in VLAN 30. Even if physically on the same switch, they behave as if on completely separate networks.

⏱️ DNS Caching & TTL

TTL (Time To Live) tells DNS resolvers how long to cache a record before checking again. Short TTL = faster propagation of changes but more DNS queries. Long TTL = fewer queries but slower change propagation.

DHCP — Automatic IP Configuration

DHCP automatically configures devices when they join a network. Watch the DORA process unfold in real time.

🤝 What is DHCP?
Without DHCP, every device on your network would need manual configuration of IP, subnet mask, gateway, and DNS — imagine doing that for 500 office computers! DHCP automates this. When a device joins a network, it broadcasts a request and the DHCP server responds with all necessary configuration. The IP is "leased" for a limited time and must be renewed.
